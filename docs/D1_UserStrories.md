User Stories & Acceptance Criteria

US-01 สมัครเข้าร่วม Workshop
As a นักศึกษา
I want สมัครเข้าร่วม Workshop ผ่านระบบ
So that สามารถเข้าร่วมกิจกรรมได้อย่างสะดวก
Acceptance Criteria
-ผู้ใช้เข้าสู่ระบบแล้ว
-เลือก Workshop และกดสมัคร
-ระบบบันทึกข้อมูลการสมัคร
-แสดงสถานะการสมัครสำเร็จ
-จำนวนที่นั่งถูกปรับลดลง
Story Point: 3
Priority: Must

US-02 ดูรายการ Workshop
As a ผู้ใช้
I want ดูรายการ Workshop ที่เปิดรับสมัคร
So that สามารถเลือกกิจกรรมที่สนใจได้
Acceptance Criteria
-แสดง Workshop ทั้งหมด
-แสดงวัน เวลา และจำนวนที่นั่ง
-แสดงสถานะ เปิด / เต็ม / ปิด
Story Point: 2
Priority: Must

US-03 ยกเลิกการสมัคร Workshop
As a นักศึกษา
I want ยกเลิกการสมัคร
So that เปิดโอกาสให้ผู้อื่นสมัครแทน
Acceptance Criteria
-ผู้ใช้สามารถยกเลิกได้
-ระบบปรับจำนวนที่นั่งกลับ
-สถานะการสมัครเปลี่ยนเป็นยกเลิก
Story Point: 2
Priority: Should

US-04 ผู้จัดสร้าง Workshop
As a ผู้จัดอบรม
I want สร้าง Workshop ใหม่
So that เปิดรับสมัครผู้เข้าร่วม
-Acceptance Criteria
-กรอกข้อมูล Workshop ได้ครบ
-กำหนดจำนวนที่นั่งได้
-กำหนดวัน เวลา และสถานะได้
Story Point: 5
Priority: Must

US-05 ตรวจสอบและอนุมัติ Workshop
As a ผู้ดูแลระบบ
I want ตรวจสอบและอนุมัติ Workshop
So that Workshop สามารถเปิดให้สมัครได้
Acceptance Criteria
-มี Workshop ที่ถูกสร้างในระบบ
-ผู้ดูแลระบบตรวจดูข้อมูล Workshop
-ผู้ดูแลระบบกดอนุมัติ
-สถานะ Workshop เปลี่ยนเป็นเปิดรับสมัคร
Workshop แสดงให้ผู้ใช้เห็น
Story Point: 3
Priority: Must

US-06 จำกัดจำนวนผู้เข้าร่วม Workshop
As a นักศึกษา
I want สมัคร Workshop ตามจำนวนที่นั่ง
So that การสมัครไม่เกินจำนวนที่กำหนด
Acceptance Criteria
-Workshop มีการกำหนดจำนวนที่นั่ง
-นักศึกษาสามารถสมัครได้เมื่อที่นั่งยังว่าง 
-เมื่อมีผู้สมัครครบ ระบบปิดรับสมัคร
สถานะ Workshop แสดงว่าเต็ม
 ไม่สามารถสมัครเพิ่มได้
Story Point: 3
Priority: Must

US-07 ตรวจสอบสถานะการสมัคร
As a นักศึกษา
I want ตรวจสอบสถานะการสมัคร Workshop
So that รู้ว่าตนเองสมัครสำเร็จหรือไม่
Acceptance Criteria
-นักศึกษาเข้าสู่ระบบ
-เปิดหน้าประวัติการสมัคร
-ระบบแสดง Workshop ที่สมัครไว้
-แสดงสถานะการสมัคร 
-แสดงวันและเวลา Workshop
Story Point: 2
Priority: Should

US-08 ดูรายชื่อผู้เข้าร่วม Workshop
As a นักศึกษา
I want ดูรายชื่อผู้เข้าร่วม Workshop
So that รู้ว่ามีใครเข้าร่วมกิจกรรมบ้าง
Acceptance Criteria
-Workshop เปิดรับสมัครแล้ว
-ระบบแสดงรายชื่อผู้เข้าร่วม
-แสดงชื่อผู้เข้าร่วมแต่ละคน
-แสดงจำนวนผู้เข้าร่วม 
Story Point: 3
Priority: Should

US-09 ส่งออกรายชื่อผู้เข้าร่วม
As a นักศึกษา
I want ส่งออกรายชื่อผู้เข้าร่วม Workshop
So that นำไปใช้ทำรายงานได้
Acceptance Criteria
-มีรายชื่อผู้เข้าร่วมในระบบ
-ผู้ใช้เลือกส่งออกรายชื่อ
-ระบบสร้างไฟล์รายชื่อ
-ไฟล์มีข้อมูลครบ
-สามารถดาวน์โหลดไฟล์ได้
Story Point: 3
Priority: Could

US-10 ดูรายงานสรุปข้อมูลการอบรม
As a นักศึกษา
I want ดูรายงานสรุปข้อมูลการอบรม
So that ใช้สรุปผลโครงงาน
Acceptance Criteria
-มีข้อมูล Workshop ในระบบ
-ผู้ใช้เปิดหน้ารายงาน
-ระบบแสดงจำนวน Workshop
-ระบบแสดงจำนวนผู้เข้าร่วม
-ข้อมูลนำไปใช้เขียนรายงานได้
Story Point: 4
Priority: Could

US-11 แก้ไขข้อมูล Workshop
As a ผู้จัดอบรม
I want แก้ไขข้อมูล Workshop
So that ข้อมูลถูกต้องก่อนวันจัดกิจกรรม
Acceptance Criteria
-ผู้จัดอบรมเข้าสู่ระบบ
-เลือก Workshop ที่ตนเองสร้างไว้
-แก้ไขข้อมูล Workshop ได้
-ระบบบันทึกข้อมูลที่แก้ไข
-ข้อมูลแสดงผลเป็นข้อมูลล่าสุด
Story Point: 3
Priority: Could

US-12 แจ้งเตือนสถานะการสมัคร
As a นักศึกษา
I want รับการแจ้งเตือนเมื่อสมัคร Workshop สำเร็จ
So that ไม่พลาดข้อมูลสำคัญเกี่ยวกับกิจกรรม
Acceptance Criteria
-นักศึกษาสมัคร Workshop สำเร็จ
-ระบบสร้างการแจ้งเตือนให้ผู้ใช้
-ผู้ใช้เห็นข้อความแจ้งเตือน
-แจ้งเตือนแสดงสถานะการสมัคร
-ข้อมูลแจ้งเตือนเข้าใจง่าย
Story Point: 2
Priority: Could
