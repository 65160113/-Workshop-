# Sprint Planning

โครงการ: ระบบลงทะเบียนอบรม / Workshop ออนไลน์

## **1. ภาพรวม Sprint**

ระยะเวลา Sprint: 2 สัปดาห์

Sprint Goal:

พัฒนาฟังก์ชันพื้นฐานของระบบ ได้แก่ สมัครสมาชิก เข้าสู่ระบบ ดูรายการ Workshop
และลงทะเบียนเข้าร่วม Workshop

## **5. Definition of Done (DoD)**

งานจะถือว่าเสร็จเมื่อ:

-   ฟังก์ชันทำงานได้ตาม User Story

-   เชื่อมฐานข้อมูลได้

-   ทดสอบแล้วไม่มี error สำคัญ

-   โค้ดถูกบันทึกใน repository

-   PO ตรวจสอบแล้ว

## **7. เหตุผลในการวางแผน Sprint**

Sprint แรกเน้นสร้าง Minimum Viable Product (MVP)

เพื่อให้ระบบสามารถทำงานได้ครบ flow หลัก:

สมัครสมาชิก → Login → ดู Workshop → ลงทะเบียน

หลังจากนั้นจึงพัฒนา:

-   ระบบ Admin

-   รายงาน

-   ฟีเจอร์เพิ่มเติม
