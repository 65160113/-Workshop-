Software Requirements Specification (SRS)
โครงการระบบลงทะเบียนอบรม / Workshop ออนไลน์ของมหาวิทยาลัย
1. บทนำ (Introduction)
1.1 วัตถุประสงค์ (Purpose)
เอกสารนี้ระบุความต้องการซอฟต์แวร์สำหรับ “ระบบลงทะเบียนอบรม / Workshop ออนไลน์ของมหาวิทยาลัย” โดยมีวัตถุประสงค์เพื่อออกแบบและพัฒนาระบบลงทะเบียนอบรม/Workshop ออนไลน์ที่ใช้งานง่าย เป็นระบบ ช่วยอำนวยความสะดวกให้ผู้เข้าร่วมสามารถสมัคร ตรวจสอบสถานะ และยกเลิกการสมัครได้ด้วยตนเอง พร้อมทั้งสนับสนุนให้ผู้จัดอบรมสามารถบริหารจัดการข้อมูล Workshop รายชื่อผู้เข้าร่วม รวมถึงรายงานและสถิติการจัดอบรมของมหาวิทยาลัยได้อย่างมีประสิทธิภาพ
1.2 ขอบเขตของระบบ (Scope)
-	ระบบแสดงรายการอบรม / Workshop และรายละเอียดกิจกรรม
-	ระบบสมัครเข้าร่วมและยกเลิกการสมัครอบรมของผู้เข้าร่วม
-	ระบบจำกัดและควบคุมจำนวนผู้เข้าร่วมในแต่ละ Workshop
-	ระบบจัดการข้อมูล Workshop โดยผู้จัดอบรม เช่น การสร้างและแก้ไขกิจกรรม
-	ระบบตรวจสอบและอนุมัติ Workshop โดยผู้อนุมัติหรือผู้ดูแลระบบ
-	ระบบแสดง ตรวจสอบ และส่งออกรายชื่อผู้เข้าร่วมกิจกรรม
-	ระบบจัดทำรายงานและสถิติสรุปข้อมูลการจัดอบรม
-	ระบบรองรับผู้ใช้งานหลายกลุ่ม ได้แก่ ผู้เยี่ยมชม นักศึกษา ผู้จัดอบรม ผู้อนุมัติ และผู้ดูแลระบบ

1.3 ภาพรวมเอกสาร (Document Overview)
      	เอกสารฉบับนี้จัดแบ่งเนื้อหาออกเป็นหัวข้อหลักเพื่อเป็นแนวทางในการพัฒนาและตรวจสอบความถูกต้อง ดังนี้
-	บทนำ: ให้ข้อมูลพื้นฐานเกี่ยวกับโครงการ วัตถุประสงค์ และขอบเขตโดยรวมของโครงการ
-	คำอธิบายโดยรวม: อธิบายบริบทและลักษณะทั่วไป
-	ความต้องการเฉพาะ: ระบุรายละเอียดความต้องการ
-	อินเทอร์เฟซภายนอก: กำหนดการเชื่อมต่อระบบ
-	ฟีเจอร์ระบบ: อธิบายฟีเจอร์แต่ละรายการ
-	 ความต้องการอื่นๆ: กฎและข้อกำหนดเพิ่มเติม 

2. คำอธิบายโดยรวม (Overall Description)
2.1 มุมมองผลิตภัณฑ์ (Product Perspective)
1.	ระบบลงทะเบียนอบรม / Workshop ออนไลน์เป็นระบบสารสนเทศแบบเว็บแอปพลิเคชันที่พัฒนาขึ้นใหม่ เพื่อใช้ภายในมหาวิทยาลัย
2.	ระบบทำหน้าที่เป็นศูนย์กลางในการจัดการข้อมูลกิจกรรมอบรม ตั้งแต่การสร้างกิจกรรม การสมัครเข้าร่วม ไปจนถึงการสรุปรายงาน
3.	ผู้ใช้งานสามารถเข้าถึงระบบผ่านเว็บเบราว์เซอร์ โดยไม่จำเป็นต้องติดตั้งโปรแกรมเพิ่มเติม
4.	ระบบรองรับผู้ใช้งานหลายบทบาท ได้แก่ ผู้เยี่ยมชม นักศึกษา ผู้จัดอบรม ผู้อนุมัติ และผู้ดูแลระบบ
5.	การทำงานของระบบเป็นไปตามขั้นตอนการจัดอบรมของมหาวิทยาลัย ตั้งแต่การเสนอ Workshop การอนุมัติ การเปิดรับสมัคร และการจัดเก็บข้อมูลผู้เข้าร่วม

2.2 ฟังก์ชันผลิตภัณฑ์ (Product Functions)
-	การลงทะเบียนและยืนยันตัวตนผู้ใช้งานระบบ
-	การเรียกดูและค้นหารายการอบรม / Workshop
-	การสมัครเข้าร่วมและยกเลิกการสมัคร Workshop
-	การควบคุมและจำกัดจำนวนผู้เข้าร่วมในแต่ละ Workshop
-	การสร้าง แก้ไข และจัดการข้อมูล Workshop (ผู้จัดอบรม)
-	การตรวจสอบและอนุมัติ Workshop ก่อนเปิดรับสมัคร (ผู้อนุมัติ / ผู้ดูแลระบบ)
-	การแสดงและส่งออกรายชื่อผู้เข้าร่วม Workshop
-	การตรวจสอบสถานะและประวัติการสมัครของผู้เข้าร่วม
-	การจัดทำรายงานและสถิติสรุปข้อมูลการจัดอบรม (ผู้ดูแลระบบ)

2.3 ประเภทผู้ใช้ (User Classes)
 ระบบลงทะเบียนอบรมออนไลน์ประกอบด้วยผู้ใช้งาน 5 กลุ่มหลัก 
1.	ผู้เยี่ยมชม (Guest)
- พื้นฐาน: บุคคลทั่วไปหรือนักศึกษาที่ยังไม่ได้เข้าสู่ระบบ
- ความถี่: ใช้งานเป็นครั้งคราว ก่อนการตัดสินใจสมัครสมาชิกหรือเข้าสู่ระบบ
- กิจกรรมหลัก: เข้าถึงหน้าเว็บไซต์เพื่อเรียกดูรายการกิจกรรมอบรม (View Workshop List) และค้นหาข้อมูลกิจกรรม (Search) ได้ แต่ไม่สามารถทำการสมัครหรือทำรายการสำคัญอื่น ๆ ได้ จนกว่าจะทำการสมัครสมาชิกและเข้าสู่ระบบ
2.	นักศึกษา (Student)
- พื้นฐาน: ผู้ใช้งานหลักของระบบที่เป็นนักศึกษาและได้ทำการเข้าสู่ระบบเรียบร้อยแล้ว
- ความถี่: ใช้งานบ่อย โดยเฉพาะในช่วงที่มีการเปิดรับสมัคร Workshop
- กิจกรรมหลัก: มีสิทธิ์ในการสมัครเข้าร่วมกิจกรรมอบรม (Enroll), ยกเลิกการสมัคร (Cancel Enrollment), และตรวจสอบประวัติการลงทะเบียนของตนเอง (View History) รวมถึงการแก้ไขข้อมูลส่วนตัว
3.	ผู้จัดอบรม (Organizer)
- พื้นฐาน: บุคลากร อาจารย์ หรือนักศึกษาที่เป็นผู้รับผิดชอบโครงการจัดอบรม
- ความถี่: ใช้งานตามรอบการจัดกิจกรรม โดยจะใช้งานบ่อยในช่วงเตรียมและดำเนินการจัดอบรม
- กิจกรรมหลัก: หน้าที่สร้างรายการกิจกรรมใหม่ (Create Workshop) โดยระบุรายละเอียดและลิงก์ห้องเรียนออนไลน์ รวมถึงสามารถตรวจสอบรายชื่อผู้ลงทะเบียน (View Attendees) เพื่อบริหารจัดการจำนวนผู้เข้าร่วม
4.	ผู้อนุมัติ (Approver)
- พื้นฐาน: เจ้าหน้าที่ฝ่ายวิชาการหรือผู้ที่มีอำนาจตัดสินใจในการเผยแพร่กิจกรรม
- ความถี่: ใช้งานเป็นครั้งคราว ตามจำนวนกิจกรรมที่ถูกส่งเข้ามาเพื่อขออนุมัติ
- กิจกรรมหลัก: หน้าที่ตรวจสอบความถูกต้องและเหมาะสมของเนื้อหากิจกรรมที่ผู้จัดส่งเข้ามา และดำเนินการเปลี่ยนสถานะกิจกรรมเป็น "อนุมัติ" (Approve) เพื่อเผยแพร่ หรือ "ไม่อนุมัติ" (Reject) หากข้อมูลไม่เป็นไปตามเกณฑ์
5.	ผู้ดูแลระบบ (Admin)
- พื้นฐาน: เจ้าหน้าที่ฝ่ายไอทีผู้ดูแลระบบหลังบ้าน
- ความถี่: ใช้งานสม่ำเสมอ หรือเมื่อมีการบำรุงรักษาและจัดการข้อมูลระบบ
- กิจกรรมหลัก: มีสิทธิ์สูงสุดในการจัดการข้อมูลพื้นฐานของระบบ (Master Data) เช่น การเพิ่ม/ลดข้อมูลคณะ (Manage Faculty), ข้อมูลแพลตฟอร์มออนไลน์ (Manage Platform) และการบริหารจัดการบัญชีผู้ใช้งาน (Manage Users) เพื่อให้ระบบดำเนินงานไปได้อย่างปกติ

3. ความต้องการเฉพาะ (Specific Requirements)
3.1 ความต้องการเชิงหน้าที่ (Functional Requirements)

REQ-001: การแสดงรายการอบรม / Workshop
ระบบจะต้องแสดงรายการอบรม / Workshop ที่เปิดให้เข้าชม โดยแสดงข้อมูลชื่อกิจกรรม วัน เวลา จำนวนที่นั่ง และสถานะการรับสมัคร เพื่อให้ผู้ใช้งานสามารถเลือกกิจกรรมที่สนใจได้
ลำดับความสำคัญ (Priority): สูง (HIGH)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-002: การสมัครเข้าร่วม Workshop
ระบบจะต้องอนุญาตให้นักศึกษาที่เข้าสู่ระบบแล้วสมัครเข้าร่วม Workshop ได้ โดยระบบต้องบันทึกข้อมูลการสมัครและปรับจำนวนที่นั่งคงเหลือโดยอัตโนมัติ
ลำดับความสำคัญ (Priority): สูง (HIGH)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-003: การยกเลิกการสมัคร Workshop
ระบบจะต้องอนุญาตให้นักศึกษายกเลิกการสมัคร Workshop ได้ โดยระบบต้องปรับจำนวนที่นั่งกลับและเปลี่ยนสถานะการสมัครเป็นยกเลิก
ลำดับความสำคัญ (Priority): กลาง (MEDIUM)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-004: การจำกัดจำนวนผู้เข้าร่วม Workshop
ระบบจะต้องควบคุมจำนวนผู้เข้าร่วม Workshop ตามจำนวนที่กำหนดไว้ และปิดรับสมัครโดยอัตโนมัติเมื่อมีผู้สมัครครบ
ลำดับความสำคัญ (Priority): สูง (HIGH)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-005: การสร้างและแก้ไข Workshop
ระบบจะต้องอนุญาตให้ผู้จัดอบรมสร้างและแก้ไขข้อมูล Workshop โดยสามารถกำหนดรายละเอียดกิจกรรม วัน เวลา จำนวนที่นั่ง และลิงก์ห้องเรียนออนไลน์ได้
ลำดับความสำคัญ (Priority): สูง (HIGH)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-006: การอนุมัติ Workshop
ระบบจะต้องรองรับกระบวนการตรวจสอบและอนุมัติ Workshop โดยผู้อนุมัติ เพื่อเปลี่ยนสถานะกิจกรรมเป็น “อนุมัติ” หรือ “ไม่อนุมัติ” ก่อนเปิดให้สมัคร
ลำดับความสำคัญ (Priority): สูง (HIGH)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-007: การแสดงและส่งออกรายชื่อผู้เข้าร่วม 
ระบบจะต้องแสดงรายชื่อผู้เข้าร่วม Workshop และสามารถส่งออกรายชื่อเป็นไฟล์ เพื่อนำไปใช้ในการจัดทำรายงานหรือเอกสารที่เกี่ยวข้อง
ลำดับความสำคัญ (Priority): กลาง (MEDIUM)
สถานะ (Status): อนุมัติแล้ว (Approved)

REQ-008: การรายงานและสถิติการจัดอบรม
ระบบจะต้องจัดทำรายงานและสถิติสรุปข้อมูลการจัดอบรม เช่น จำนวน Workshop และจำนวนผู้เข้าร่วม เพื่อสนับสนุนการจัดทำรายงานของมหาวิทยาลัย
ลำดับความสำคัญ (Priority): ต่ำ (LOW)
สถานะ (Status): อนุมัติแล้ว (Approved)

3.2 ความต้องการที่ไม่ใช่เชิงหน้าที่ (Non-Functional Requirements)
NFREQ-001: เวลาในการตอบสนองของระบบ
ระบบควรแสดงผลหน้าจอหลักและรายการ Workshop ภายในระยะเวลาไม่เกิน 3 วินาที ภายใต้การใช้งานปกติ
ลำดับความสำคัญ (Priority): สูง (HIGH)
NFREQ-002: ประสิทธิภาพในการประมวลผลการสมัคร Workshop
ระบบต้องสามารถบันทึกข้อมูลการสมัครหรือยกเลิกการสมัคร Workshop และปรับจำนวนที่นั่งได้อย่างถูกต้องแบบทันที
ลำดับความสำคัญ (Priority): สูง (HIGH)
NFREQ-003: ความปลอดภัยของข้อมูลผู้ใช้งาน
ระบบต้องจัดเก็บข้อมูลส่วนบุคคลของผู้ใช้งานอย่างปลอดภัย และป้องกันการเข้าถึงหรือแก้ไขข้อมูลโดยไม่ได้รับอนุญาต
ลำดับความสำคัญ (Priority): สูง (HIGH)
NFREQ-004: ความปลอดภัยของข้อมูลผู้ใช้งาน
 ระบบต้องจัดเก็บข้อมูลส่วนบุคคลของผู้ใช้งานอย่างปลอดภัย และป้องกันการเข้าถึงหรือแก้ไขข้อมูลโดยไม่ได้รับอนุญาต
ลำดับความสำคัญ (Priority): สูง (HIGH)

4.ความต้องการด้านส่วนติดต่อภายนอก (External Interface Requirements)
4.1  อินเทอร์เฟซฮาร์ดแวร์ (Hardware Interfaces)
a) เว็บไซต์สำหรับผู้ใช้งานทั่วไป (User / Student Interface)
- รูปแบบการออกแบบ: Responsive Web Design
- เทคโนโลยีที่ใช้: HTML5, CSS3, JavaScript, React.js, Tailwind CSS, Node.js, Express.js
- อุปกรณ์ที่รองรับ: คอมพิวเตอร์ตั้งโต๊ะ / โน้ตบุ๊ก, แท็บเล็ต, สมาร์ตโฟน
- เบราว์เซอร์ที่รองรับ: Google Chrome, Mozilla Firefox, Microsoft Edge, Safari (เวอร์ชันล่าสุด 2 เวอร์ชัน)
หน้าจอหลักที่ต้องมี
- หน้าแรก (Home)
- รายการ Workshop (Workshop List)
- รายละเอียด Workshop (Workshop Detail)
- สมัคร / ยกเลิกการสมัคร (Enroll / Cancel)
- ประวัติการสมัคร (Enrollment History)
ลักษณะการแสดงผล
- แสดง Workshop ในรูปแบบ Card หรือ Table
- แสดงสถานะกิจกรรม: เปิดรับสมัคร / เต็ม / ปิดรับสมัคร
-  มีข้อความแจ้งเตือนเมื่อทำรายการสำเร็จหรือผิดพลาด

b) แผงควบคุมสำหรับผู้จัดอบรม (Organizer Dashboard)
รูปแบบการออกแบบ: Web Dashboard

4.2  อินเทอร์เฟซซอฟต์แวร์ (Software Interfaces)
  ฝั่งผู้ใช้งาน (Client Side)

-  หน่วยความจำขั้นต่ำ: 2 GB RAM
-  การเชื่อมต่ออินเทอร์เน็ต: อย่างน้อย 1 Mbps

-  Database Server สำหรับจัดเก็บข้อมูล
-  ไม่จำเป็นต้องใช้อุปกรณ์ฮาร์ดแวร์เฉพาะทางเพิ่มเติม

4.3  อินเทอร์เฟซการสื่อสาร (Communication Interfaces)

ใช้สำหรับ : แจ้งผลการสมัคร, แจ้งการยกเลิก, แจ้งผลการอนุมัติ Workshop
c) ระบบจัดการไฟล์ (File Export Service)
รองรับการส่งออกข้อมูลเป็น : Microsoft Excel (.xlsx), PDF (.pdf)
ใช้สำหรับ : รายชื่อผู้เข้าร่วม Workshop, รายงานสรุปกิจกรรม

5.ฟีเจอร์ระบบ (SYSTEM FEATURES)
5.1 ฟีเจอร์ที่ 1: ระบบจัดการ Workshop
คำอธิบาย
 ระบบช่วยให้ผู้จัดอบรมสามารถสร้าง แก้ไข และจัดการกิจกรรม Workshop ได้อย่างเป็นระบบ
รายละเอียดการทำงาน
-  ผู้จัดสามารถเพิ่ม Workshop ใหม่
- กำหนดรายละเอียด
- ชื่อกิจกรรม
- วัน เวลา
- จำนวนที่นั่ง
- รูปแบบกิจกรรม (ออนไลน์/ออนไซต์)
-  แก้ไขข้อมูล Workshop ได้ก่อนวันเริ่มกิจกรรม
-  เปลี่ยนสถานะ Workshop เป็น
- รออนุมัติ
- อนุมัติ
- ไม่อนุมัติ

5.2 ฟีเจอร์ที่ 2: ระบบสมัครและยกเลิกการสมัคร
คำอธิบาย
 ระบบอำนวยความสะดวกให้นักศึกษาสามารถสมัครและยกเลิกการสมัคร Workshop ได้ด้วยตนเอง
รายละเอียดการทำงาน
-  นักศึกษาต้องเข้าสู่ระบบก่อนทำรายการ
-  เลือก Workshop และกดสมัคร
-  ระบบตรวจสอบจำนวนที่นั่งอัตโนมัติ
-  เมื่อนั่งเต็ม ระบบจะปิดรับสมัคร
-  นักศึกษาสามารถยกเลิกการสมัครได้ก่อนวันกิจกรรม
-  ระบบปรับจำนวนที่นั่งโดยอัตโนมัติ

5.3 ฟีเจอร์ที่ 3: ระบบอนุมัติ Workshop
คำอธิบาย
 ระบบรองรับกระบวนการตรวจสอบและอนุมัติกิจกรรมโดยผู้มีอำนาจ
รายละเอียดการทำงาน
-  ผู้อนุมัติเข้าดูรายการ Workshop ที่รอพิจารณา
-  ตรวจสอบข้อมูลความเหมาะสมของกิจกรรม
-  เลือก : อนุมัติ (Approve) หรือ ไม่อนุมัติ (Reject)
-  ระบบแจ้งผลการพิจารณาไปยังผู้จัดอบรม
-  เฉพาะ Workshop ที่ผ่านการอนุมัติเท่านั้นที่จะแสดงให้ผู้ใช้สมัครได้

5.4 ฟีเจอร์ที่ 4: ระบบรายงานและส่งออกข้อมูล
คำอธิบาย
 ระบบช่วยสรุปข้อมูลและจัดทำรายงานสำหรับการบริหารจัดการ
รายละเอียดการทำงาน
    แสดงรายงาน
-  จำนวน Workshop ทั้งหมด
- จำนวนผู้เข้าร่วมแต่ละกิจกรรม
      แอดมินสามารถ
- ดูรายชื่อผู้เข้าร่วม
-  ส่งออกข้อมูลเป็นไฟล์ Excel / PDF
     ใช้ข้อมูลสำหรับ
-  รายงานผลโครงการ
-  สรุปผลงานประจำปีของมหาวิทยาลัย
