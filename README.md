ภาพรวมระบบลงทะเบียนอบรมออนไลน์ (Project Overview)
โครงการ "ระบบลงทะเบียนอบรมและเวิร์กชอปออนไลน์ (University Workshop Registration System)" พัฒนาขึ้นเพื่อแก้ปัญหาความยุ่งยากในการจัดการเอกสารและการสมัครแบบเดิม โดยเปลี่ยนมาใช้แพลตฟอร์มเว็บแอปพลิเคชันที่เป็นศูนย์กลาง (Centralized Platform) ในการบริหารจัดการข้อมูลกิจกรรมทั้งหมด

ระบบถูกออกแบบให้รองรับผู้ใช้งาน 5 กลุ่มหลัก ซึ่งมีบทบาทและหน้าที่สัมพันธ์กันเป็นวงจร (Ecosystem) ดังนี้:

1. จุดเริ่มต้น: การสร้างและตรวจสอบ (Creation & Approval)
กระบวนการเริ่มจาก ผู้จัดอบรม (Organizer) ซึ่งเป็นอาจารย์หรือบุคลากร ทำหน้าที่สร้างโครงการใหม่ในระบบ โดยระบุรายละเอียด วิทยากร และจำนวนที่นั่ง จากนั้นข้อมูลจะถูกส่งต่อไปยัง ผู้อนุมัติ (Approver) เพื่อทำการตรวจสอบความถูกต้องของเนื้อหา หากผ่านเกณฑ์ กิจกรรมจะถูกเปลี่ยนสถานะเป็น "อนุมัติ" และเริ่มแสดงผลบนหน้าเว็บไซต์ทันที

2. การเข้าถึงและการสมัคร (Access & Enrollment)
ในฝั่งผู้ใช้งานทั่วไป เริ่มต้นในฐานะ ผู้เยี่ยมชม (Guest) ซึ่งสามารถเข้ามาค้นหาและดูรายละเอียดกิจกรรมที่สนใจได้ แต่หากต้องการสมัครเข้าร่วม จะต้องเข้าสู่ระบบเพื่อยืนยันตัวตนเป็น นักศึกษา (Student) ก่อน เมื่อเป็นนักศึกษาแล้ว จะสามารถกดลงทะเบียน (Enroll), ตรวจสอบสถานะที่นั่งแบบ Real-time, และดูประวัติการเข้าร่วมกิจกรรมของตนเองได้ รวมถึงสามารถยกเลิกสิทธิ์ได้หากไม่สะดวกเข้าร่วม เพื่อคืนโควตาที่นั่งให้ผู้อื่น

3. การบริหารจัดการและควบคุม (Management & Control)
เบื้องหลังการทำงานทั้งหมดจะมี ผู้ดูแลระบบ (Admin) คอยบริหารจัดการข้อมูลพื้นฐาน (Master Data) เช่น ข้อมูลคณะ ข้อมูลแพลตฟอร์มออนไลน์ และบัญชีผู้ใช้งาน เพื่อให้ระบบมีความเสถียรและพร้อมใช้งาน รวมถึงการออกรายงานสรุปผลภาพรวมเพื่อนำไปวิเคราะห์แนวโน้มความสนใจของนักศึกษาต่อไป

สรุปความสัมพันธ์ของ 5 ตัวละคร (Actor Relationship)
Organizer (ผู้จัดอบรม): "ฉันมีกิจกรรมดีๆ อยากจัด" -> สร้าง Workshop
Approver (ผู้อนุมัติ): "ขอดูก่อนว่าเหมาะสมไหม" -> อนุมัติให้เผยแพร่
Guest (ผู้เยี่ยมชม): "มีอะไรน่าสนใจบ้างนะ" -> ค้นหาและดูรายละเอียด
Student (นักเรียน): "อันนี้น่าสนใจ ลงชื่อเลยละกัน" -> สมัคร/ยกเลิก/ดูประวัติ
Admin (ผู้ดูแล): "ระบบต้องเรียบร้อย ข้อมูลต้องเป๊ะ" -> จัดการหลังบ้าน
